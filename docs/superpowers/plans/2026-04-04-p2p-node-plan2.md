# GOAMP P2P Node — Plan 2: P2P Layer

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Prerequisite:** Plan 1 complete and binary builds.

**Goal:** Add real libp2p networking — QUIC+TCP transport, Kademlia DHT catalog, GossipSub profile sync, P2P stream handlers, and a test utility for spinning up N-node networks.

**Architecture:** The `sdk/node/` package gains a real libp2p host. Each SDK module (catalog, profiles) gets a P2P layer that runs alongside its SQLite layer — local operations stay local, P2P operations propagate to the network. testutil.NewNetwork enables integration tests without any real sockets.

**Tech Stack:** go-libp2p v0.35+, go-libp2p-kad-dht, go-libp2p-pubsub, go-libp2p/p2p/discovery/mdns, Noise protocol, QUIC transport.

### Task 16 — 🤖 CLAUDE: Kademlia DHT + DNS bootstrap

**Files:** `sdk/node/dht.go`, `sdk/node/bootstrap.go`, `sdk/node/mdns.go`

`dht.go` — initialise `go-libp2p-kad-dht` in client mode for `--mode=client`, server mode for `--mode=full`/`server`.

`mdns.go` — `mdns.NewMdnsService(h, "_goamp._tcp.local", notifee)` where notifee calls `h.Connect`.

### Task 19 — 👤 YOU: /goamp/sync/1.0

**File:** `sdk/catalog/sync_handler.go`

Incremental sync: when a new peer connects, we offer our catalog in batches.

```
Initiator sends: SyncRequest { since_unix_ts: int64 }
Responder streams: repeated SyncBatch { tracks: []Track, has_more: bool }
```

Add `SyncRequest` and `SyncBatch` to `proto/goamp.proto` (and regenerate, or hand-write the Go struct matching the proto layout).

The handler reads all tracks from the store with `created_at > since_unix_ts`, sends them in batches of 100, then sends a final `SyncBatch{has_more: false}`.
